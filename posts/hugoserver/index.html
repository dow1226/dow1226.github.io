前言#

因為工作關係，總是要學習多領域的知識，但因為沒有筆記的習慣，所以常常會在一段時間後忘記所學習到的內容 因此，想嘗試用Blog的方式來當作筆記紀錄，除了讓自己好搜尋外，一方面當作網頁技術的學習練習。

hugo PaperMod 主題安裝#

- 建立一個存放 hugo web 網站的資料夾 並且使用 yaml格式
> hugo new site <blog_name> -f yaml

- 到資料夾內用git clone下載 PaperMod主題
> git clone https://github.com/adityatelange/hugo-PaperMod themes/PaperMod --depth=1

- 建立新文章
> hugo new post posts\<文章名稱>\index.md

- 預覽網頁效果
> hugo server -D

在預設狀態下打開首頁看不到所建立的文章 需要到 hugo.yaml 做調整 不知道為什麼 網路上找到的文章都是說修改 config.yml 但總之我是沒有產生這個檔案，所以我將 PaperMod的範例文件直接複製過來

文章預設格式設定#

文章的預設格式設定需要新建一個資料夾並將模板文章放進去 設定方式參考自https://www.lilmp.com/2022-06-22/hugo-papermod-blog/#papermod-%e4%b8%bb%e9%a1%8c

產生模板文章檔案
> hugo new --kind post archetypes/default.md

---
---
title: "{{ replace .Name "-" " " | title }}"
date: {{ .Date }}
draft: false

summary: "" # 首頁顯示的文字總結段落

categories: # 自由新增
- 
- 
tags: # 自由新增
- 
- 

author: ["作者名稱"]

isCJKLanguage: true # 是否是中文(chinese,japanese,korea) 字數判斷用
comments: true
showToc: true # 顯示目錄
TocOpen: true # 預設打開目錄
hidemeta: false # 是否隱藏meta訊息(ex:發布日期、作者...etc)
disableShare: false # 取消社群分享區塊
showbreadcrumbs: true # 於頂部顯示文章路徑
ShowWordCounts: true
ShowReadingTime: true
ShowLastMod: true

cover:
    image: "" # 封面圖片路径 (ex:clear-301-redirection-cache-chrome/cover.jpg)
    caption: "" # 封面圖片底部描述
    alt: ""
    relative: false
---

當產生文章時會自動套用這個模板 但其實不用模板也可以新建文章 只要符合 Markdown語法就行 就像以下文章範例

+++
title = 'Helloworld'
date = 2024-09-28T16:13:31+08:00
draft = true
+++


Hello World

產生靜態網站輸出#

當文章編輯完成後使用以下語法將當前設定輸出

> hugo

在資料夾下會產生 public 的資料夾 資料夾內為hugo所產生的靜態網站檔案 將其複製貼上到欲部屬的位置即可完成部屬 如本網站部屬在 github上，點進連結即可看到有關於本 blog 的所有設定檔案

不過直接把部屬網站全部公開還是有點資安疑慮啦，但初次學習先簡單部屬，等熟悉後再來深化相關知識吧
